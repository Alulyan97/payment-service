# Payment Service

Сервис для создания счетов и приема платежей через вебхуки.

## Как запустить проект

1. **Склонировать репозиторий**

```bash
git clone https://github.com/Alulyan97/payment-service.git
cd payment-serviceУстановить зависимости

bash
npm install
Настроить переменные окружения

Создай файл .env в корне проекта и добавь туда:

text
PORT=3000
MONGODB_URI=строка_подключения_к_mongodb
REDIS_URL=строка_подключения_к_redis
MongoDB можно взять бесплатно на mongodb.com/atlas

Redis — на redis.com/cloud

Запустить сервер

bash
npm run dev
Сервер запустится на http://localhost:3000

Как запустить тесты
bash
npm test
Будут проверены расчет комиссии,работа подписи

Как проверить API через Postman
Создать счет
POST http://localhost:3000/api/invoice

Тело запроса:

json
{
    "amount": 1000,
    "merchantId": "test_merchant"
}
Получить счет
GET http://localhost:3000/api/invoice/{invoiceId}

Отправить вебхук об оплате
POST http://localhost:3000/api/webhook

Тело:

json
{
    "invoiceId": "тот_который_вернулся_при_создании",
    "status": "paid"
}
Допущения, которые я принял в работе
Комиссия продавца сделал фиксированной 2.5%. В реальном проекте она бралась бы из настроек продавца
При создании счета в БД автоматически создается тестовый продавец test_merchant.

Nonce хранится в Redis 24 часа, чтобы защититься от повторных запросов.

Если Redis недоступен, проверка nonce пропускается но в реальном проекте без Redis нельзя

Timestamp проверяется на актуальность примерно 5 минут чтобы отсеивать старые запросы

Что можно улучшить, если бы было больше времени

Логирование, писать все важные действия в файл или в отдельную коллекцию БД
Баланс продавца хранить, сколько денег уже поступило
Мониторинг, добавить эндпоинт /health и собирать статистику по платежам.
Документация API — например, через Swagger.
